#### 02. 독서 궁합 측정

**위치**: `/:username` (공개 책장 페이지)

**요구사항**:

```
[ ] 1. DB 스키마 설계
    ├─ compatibility_results 테이블 생성
    │   ├─ id (uuid, PK)
    │   ├─ user_a_id (uuid, FK → profiles)
    │   ├─ user_b_id (uuid, FK → profiles)
    │   ├─ compatibility_score (integer, 0-100)
    │   ├─ personality_comparison (jsonb)
    │   │   ├─ user_a_type (string)
    │   │   ├─ user_b_type (string)
    │   │   ├─ similarities (array)
    │   │   └─ differences (array)
    │   ├─ genre_similarity (jsonb)
    │   │   ├─ common_genres (array)
    │   │   ├─ similarity_percentage (number)
    │   │   └─ insights (string)
    │   ├─ mutual_recommendations (jsonb)
    │   │   ├─ for_user_a (array of books)
    │   │   └─ for_user_b (array of books)
    │   ├─ created_at (timestamp)
    │   └─ updated_at (timestamp)
    └─ RLS 정책 설정 (SELECT: 본인만 조회 가능)

[ ] 2. API 구현
    ├─ POST /api/compatibility/analyze
    │   ├─ Request Body: { targetUserId: string }
    │   ├─ 나와 상대방의 완독한 책 목록 조회
    │   ├─ 각자의 별점 데이터 가져오기
    │   ├─ 알라딘 API로 책 메타데이터 가져오기
    │   │   └─ 제목, 저자, 카테고리, 책 소개, 목차
    │   ├─ OpenAI API 호출 (GPT-4o-mini)
    │   │   ├─ 프롬프트: 두 사람의 독서 성향 비교 분석
    │   │   ├─ JSON 모드 사용
    │   │   └─ 응답: 궁합 점수, 비교 분석, 추천 도서
    │   ├─ DB에 결과 저장 (캐싱)
    │   └─ Response: CompatibilityResult
    └─ GET /api/compatibility/[user_a_id]/[user_b_id]
        ├─ 캐싱된 결과 조회
        └─ 없으면 null 반환

[ ] 3. AI 프롬프트 설계
    ├─ 시스템 프롬프트
    │   ├─ "두 사람의 독서 성향을 비교 분석하는 전문가"
    │   ├─ 한국어 응답
    │   └─ JSON 구조화된 응답
    ├─ 사용자 프롬프트
    │   ├─ 사용자 A 독서 목록 (제목, 저자, 카테고리, 별점, 책 소개)
    │   ├─ 사용자 B 독서 목록 (동일)
    │   └─ 분석 요청 항목
    │       ├─ 궁합 점수 (0-100)
    │       ├─ 독서 성향 비교
    │       ├─ 공통 관심사 & 차이점
    │       └─ 서로에게 추천할 책 (각 3권)
    └─ 응답 형식 (TypeScript 타입)
        └─ CompatibilityAnalysis 인터페이스

[ ] 4. UI 구현
    ├─ 공개 책장 페이지에 "독서 궁합 측정하기" 버튼 추가
    │   └─ 본인 책장에서는 버튼 숨김
    ├─ CompatibilityResultPage 컴포넌트 생성
    │   ├─ 궁합 점수 카드 (0-100%, 시각화)
    │   ├─ 궁합 타입 표시 (예: "서로 다른 세계", "영혼의 책벗")
    │   ├─ 독서 성향 비교
    │   │   ├─ 나의 성향
    │   │   ├─ 상대방 성향
    │   │   └─ 공통점 & 차이점
    │   ├─ 장르 유사도 차트
    │   └─ 상호 추천 도서 (각 3권, 카드 형태)
    ├─ 로딩 상태 UI
    │   └─ "AI가 두 분의 독서 취향을 분석 중입니다..."
    └─ 에러 처리
        ├─ 책이 없는 경우
        └─ AI 분석 실패 시

[ ] 5. 캐싱 전략
    ├─ DB에 결과 저장 (영구 캐싱)
    ├─ 재분석 조건
    │   ├─ 수동 재분석 버튼 클릭
    │   └─ 7일 경과 시 재분석 권장 메시지
    └─ 비용 절감 (같은 쌍은 한 번만 분석)

[ ] 6. 테스트
    ├─ 두 사용자 간 궁합 측정 플로우
    ├─ AI 응답 파싱 테스트
    ├─ 캐싱 동작 확인
    └─ 알라딘 API 연동 확인
```

**예상 시간**: 6-8시간

**AI 프롬프트 예시**:

```typescript
// system prompt
"당신은 두 사람의 독서 성향을 비교 분석하는 전문가입니다.
책 제목, 저자, 카테고리, 별점, 책 소개를 바탕으로
두 사람의 독서 궁합을 0-100점으로 평가하고,
구체적인 비교 분석과 상호 추천 도서를 제시하세요."

// user prompt
"사용자 A: [책 목록]
사용자 B: [책 목록]

다음 항목을 분석해주세요:
1. 궁합 점수 (0-100)
2. 각자의 독서 성향 (한 문장)
3. 공통 관심사 & 차이점
4. A에게 추천할 B의 책 3권 (이유 포함)
5. B에게 추천할 A의 책 3권 (이유 포함)"
```
