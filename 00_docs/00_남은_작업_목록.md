### A. 마이그레이션 SQL 적용 — 순위 변동이 이것 없이는 안 뜬다

**파일**: `supabase/migrations/20260713_create_ranking_snapshots.sql`
**방법**: Supabase 대시보드 → SQL Editor → 파일 내용 붙여넣고 Run

> ⚠️ 로컬에 Postgres가 없어 **문법을 실행 검증하지 못했다.** 에러가 나면 그대로 알려줄 것.
> plpgsql의 `RETURNS TABLE` 출력명이 테이블 컬럼과 겹쳐 `ambiguous`로 터지는 함정은
> `#variable_conflict use_column` 으로 막아 뒀다.

**적용 후 확인**:
```sql
-- 1) 함수가 생겼는지
select proname from pg_proc
where proname in ('snapshot_book_rankings', 'get_book_ranking_with_delta');

-- 2) 첫 스냅샷을 수동으로 한 번 찍는다 (cron이 돌 때까지 기다리지 않아도 된다)
select * from public.snapshot_book_rankings();

-- 3) 다음 날 이후에 delta가 붙는지
select isbn, rank, rank_delta, is_new, has_history
from public.get_book_ranking_with_delta('most', 5);
```

> 스냅샷이 **최소 하루치** 쌓여야 변동이 계산된다. 오늘 찍고, 내일부터 `▲12`가 뜬다.
> 그 전까지는 `has_history=false` → 뱃지가 그려지지 않는다(정직하게 침묵).

### B. Vercel 환경변수

```
CRON_SECRET       = <임의의 긴 문자열>   # cron 엔드포인트 보호. 없으면 누구나 호출 가능
OPENAI_API_KEY    = sk-...              # 이미 있다면 확인만
ALADIN_API_KEY    = <TTB 키>            # NEXT_PUBLIC_ 접두사 없이! (1dbaf3b 에서 서버 전용화됨)
SUPABASE_SERVICE_ROLE_KEY = <키>        # cron 이 스냅샷을 쓸 때 필요
```

> ⚠️ 위 4개에 **`NEXT_PUBLIC_` 접두사를 붙이면 안 된다.** 붙이는 순간 Next.js가
> 클라이언트 번들에 그대로 인라인한다. service_role 키는 RLS를 우회하는 마스터 키다.
> (로컬 `.env`는 2026-07-13에 교정했다. 백업: `.env.backup-20260713`)

### C. 로컬 빌드 주의

`.env`의 `OPENAI_API_KEY`가 **빈 값**이다(키는 Vercel에만 두기로 결정).
`src/shared/lib/openai/client.ts`가 모듈 로드 시점에 throw하므로 로컬 빌드는 이렇게 돈다:

```bash
OPENAI_API_KEY=dummy npx next build
```

로컬에서 AI 분석(취향·궁합)을 실제로 테스트하려면 진짜 키를 `.env`에 넣어야 한다.

## 📎 참고: 오늘 확립된 원칙

Phase 1~3에서 정한 것. 새 화면을 만들 때 지킨다.

1. **그림자를 쓰지 않는다.** 입체는 `1px solid var(--line)`으로 만든다.
   그림자는 실제로 떠 있어야 하는 것(dialog·dropdown·popover·toast)에만.
2. **표지를 크롭하지 않는다.** 높이(또는 너비)만 고정하고 나머지는 원본 비율대로.
   판형의 불균일이 "진짜 책"의 증거다.
3. **letter-spacing을 건드리지 않는다.** Pretendard는 자간이 이미 최적화돼 있다.
4. **이모지를 헤딩·라벨에 쓰지 않는다.** lucide 단색 아이콘 또는 아이콘 없음.
5. **"당신"을 쓰지 않는다.** 주어를 생략하거나 `{닉네임}님`. AI 프롬프트에도 금지 지침이 있다.
6. **본문을 흐리게 칠하지 않는다.** 읽으라고 쓴 글은 `--text-strong` / `--text-body`.
7. **유채색은 직무를 하나씩 가진다.** 딥그린=완독, 테라코타=순위 상승, destructive=삭제.
8. **채워진 액센트 버튼은 화면당 최대 1개.**
9. **집계 화면에는 기준일을 박는다.** 날짜를 박는다는 건 갱신 책임을 진다는 선언이다.
10. **지어내지 않는다.** 업데이트 로그는 git 이력에서, 개인정보처리방침은 코드에서 확인한 것만.
