## 0. 한 줄 요약

문제는 "디자인이 촌스럽다"가 아니다. **화면에 시간·화자·도메인의 흔적이 없다.** 그래서 아무리 다듬어도 "방금 생성된 목업"으로 읽힌다. 고칠 것은 색이 아니라 **화면이 무엇을 증언하는가**다.

### 1.2 표면 증상 12가지

1. **인디고 `#6366f1`** — shadcn 기본에서 반 발짝 옮긴 색. AI 랜딩의 시그니처.
2. **이모지를 섹션 아이콘으로 사용** — 🔥📚🤖🎯📖⭐💚🌱📬🎁🚀💯. 두 벤치마크 모두 헤딩에 이모지를 쓰지 않는다. **가장 강한 AI 신호.**
3. **전면 중앙 정렬** — 히어로·섹션 제목·부제·카드 전부.
4. **3개 피처 카드 그리드** — 랜딩 템플릿 클리셰 1순위.
5. **"당신" 남발** — 히어로·부제·분석 카드·푸터.
6. **모든 섹션이 "제목 + 부제" 쌍**으로 기계적 반복.
7. **대시보드만 영어** — `Dashboard` / `Books Read` / `Overview of your reading journey`.
8. **약관·개인정보·소개·문의가 전부 "🚧 준비 중"** — "이건 미완성 데모"라는 자백.
9. **파비콘이 Next.js 기본** — 초기 커밋 이후 한 번도 안 바뀜.
10. **`public/`에 `create-next-app` 데모 SVG 5개뿐** (`next.svg`, `vercel.svg`…). 브랜드 자산 0개.
11. **`/images/placeholder-cover.png` 가 404** — `public/images/` 폴더가 없다. 표지 없는 책은 깨진 이미지. **실제 버그.**
12. **디자인 토큰 3중 분열** — `packages/design-tokens`(하늘색, **어디서도 import 안 됨**) / `globals.css`(인디고, 실사용) / `05_대시보드_UI_UX_명세.md`(에메랄드).

### 1.3 근본 원인 — 결핍 3가지

표면을 다 고쳐도 이 셋이 없으면 여전히 목업이다.

#### ① 시간의 부재

두 벤치마크는 **날짜를 문장 안에 박아둔다.**

| 교보 | 밀리 |
| --- | --- |
| `▲448 급상승` | `▲12` `▼1` `NEW` |
| `7/22(수) 출고예정` | `07월 13일 월요일 업데이트` |
| `2026.07.12` (집계 기준) | `2026.07.12` (랭킹 기준) |
| `접수 기간 : 6월 18일(목) ~ 8월 4일(화)` | `공개 D-1` · `매주 금요일 신간 공개!` |
| | `첫 달 무료 혜택 종료 사전 안내_08/04(화)` |
| | `씨네21 No.1565 : 2026.07.21` |

`▲448`은 **어제 스냅샷이 DB에 있어야** 계산되고, `7/22(수)`는 **오늘이 며칠인지 아는 백엔드**가 있어야 나온다.
**AI 카피는 시간을 못 박는다 — 박으면 틀리기 때문이다. 날짜를 박는다는 건 누군가 갱신 책임을 지고 있다는 선언이다.**

page0127 랜딩에는 날짜도, 변동도, "언제 기준"인지도 없다.

#### ② 화자의 부재

| 서비스 | 섹션 제목의 주어 |
| --- | --- |
| 교보 | "**MD들이** 신중하게 골랐어요" · "**출판사에서** 자신있게 추천해요" · "**사람들이** 많이 찾고 있어요" · "**교보문고가** 만들었어요" |
| 밀리 | "**밀리가** 주목한 책" · "**밀리에서** 직접 기획하고 만든" · "**밀리가** 독자께 바치는 헌사" · "**밀리의** 발견" |
| **page0127** | "이런 이야기를 들려드려요" — **주어 없음** |

주어가 사용자("당신")가 아니라 **화자**다. AI 슬롭은 정반대로 주어가 항상 사용자이고 화자가 없다 — *"당신의 독서 여정을 시작하세요"* 에는 **말하는 사람이 없다.**

실제로 밀리는 2인칭 "당신"을 홈 전체에서 **3개 캠페인, 4회**만 쓴다. **섹션 제목·UI 문구에는 0회.** page0127은 히어로·부제·카드·푸터·분석 결과까지 전부 "당신"이다.

#### ③ 도메인의 부재

- **교보**: 카드 radius `16px 16px 16px 0px` — 좌하단만 직각. **44회 반복.** 책등(spine) 셰이프다.
- **밀리**: 표지 radius **7px** — 그런데 토큰에는 `4/6/8/10/12/14/16/20/28/32`만 있고 **7은 없다.** 표지만 토큰 밖에서 하드코딩.
- **밀리(결정적)**: "밀리의 발견" 섹션의 표지는 **width만 125px 고정, height는 175/177/183/184/185/187/188/190px 제각각**이다. → **원본 판형을 유지하고 크롭하지 않는다.** 책마다 실제 판형이 다르다는 걸 **굳이 살려뒀다.**

page0127은 책 서비스인데 화면 어디에도 책의 물성이 없다. radius 12px, pill, 그림자 — 어떤 SaaS에나 붙일 수 있는 껍데기다.

> **한 문장**: 실서비스는 *시간*과 *화자*와 *도메인*의 흔적을 화면에 남긴다. AI 목업은 이 셋을 가질 수 없다 — 그래서 토큰이 지나치게 깨끗하고, 헤드라인이 지나치게 크고, 모든 카드가 똑같이 친절하다.

## 2. 디자인 시스템 재설정

### 2.1 컬러 — 인디고를 버린다

**두 벤치마크의 공통 원리: 무채색이 지배하고, 유채색은 "장식"이 아니라 "직무"다.**

- 교보: 텍스트 노드의 **85%가 무채색**. 그린=할인율, 레드=급상승 **전용**, 인디고=구매 버튼.
- 밀리: 브랜드 노랑(`#ffeb60`)이 **면적 상위 15위 안에 없다. 로고에만 산다.** 컬러는 콘텐츠 타입 분류용 — 오디오북 `#a451f7`, 챗북 `#eb683f`, 랭킹 변동 `#ff5b4f`.
- **밀리는 채워진 컬러 CTA 버튼이 0개다.** 주 CTA("첫 달 무료 시작하기")조차 `#161616` 패널 안의 **흰 버튼**이다.

page0127의 정체성은 **개인 서재 · 기록 · 취향**. 종이와 잉크로 간다.

```css
:root {
  /* 배경 — 순백 대신 종이 */
  --bg-page:    #faf9f7;
  --bg-surface: #ffffff;
  --bg-sunken:  #f4f2ee;

  /* 텍스트 4단계 — 순검정은 쓰지 않는다 (밀리 #242424 방식) */
  --text-1: #1f1f1f;   /* 제목·책 제목 */
  --text-2: #4a4a4a;   /* 본문 */
  --text-3: #6f6f6f;   /* 저자·출판사·메타 */
  --text-4: #9a9a9a;   /* 캡션 */

  /* 경계 — 그림자 대신 선 */
  --line:      #e5e2dc;
  --line-soft: #f0ede8;

  /* 유채색 — 각각 직무 하나씩 */
  --accent:    #2f5d3a;  /* 딥그린: 완독·목표 달성에만 */
  --accent-bg: #eef2ed;
  --rank-up:   #c25e3a;  /* 테라코타: 순위 상승에만 */
}
```

**색의 직무 고정**

| 색 | 오직 이것에만 | 쓰지 말 것 |
| --- | --- | --- |
| `--accent` 딥그린 | 완독 표시, 목표 달성 | 링크, 일반 버튼 |
| `--rank-up` 테라코타 | 순위 상승 `▲12` | 삭제, 에러 |
| `--destructive` | 삭제·탈퇴 | 그 외 전부 |

**두 가지 핵심 규칙**

1. **본문에 `muted-foreground`를 쓰지 않는다.** 지금 랜딩에서 가장 많이 쓰인 텍스트 색이 `#64748b`(19회)다. **읽으라고 쓴 글을 흐리게 칠해놨다.**
2. **채워진 액센트 버튼은 화면당 최대 1개.** 나머지는 흰 배경 + 1px 보더. (밀리 0개 / 교보는 커머스라 구매 버튼만)

### 2.2 타이포그래피 — Pretendard, 스케일 6개

```css
--font-sans: 'Pretendard Variable', Pretendard, -apple-system,
             'Apple SD Gothic Neo', sans-serif;
```

> **⚠️ 정정: `letter-spacing`을 건드리지 말 것.**
> 처음엔 교보를 따라 `-0.01em`을 권했으나, 실측 결과 **밀리는 Pretendard를 쓰면서 전 요소 `letter-spacing: normal`**이다. 교보가 `-0.01em`을 쓰는 건 NotoSansKR/Roboto 스택이라서다. **Pretendard는 자간이 이미 최적화돼 있어 추가 조정이 오히려 해롭다.** 폰트를 Pretendard로 바꾸면 자간은 손대지 않는다.

**스케일 6개로 못 박는다** (교보 실사용 6종·상위 3개가 99% / 밀리 토큰은 다단이나 실사용은 14·12·20px에 집중)

| 토큰 | size / line-height | weight | 용도 |
| --- | --- | --- | --- |
| `display` | 28 / 40 | 700 | 랜딩 h1 (**50px → 28px**) |
| `heading` | 20 / 30 | 700 | 섹션 제목 (밀리도 20px) |
| `body` | 16 / 24 | **500** | 책 제목 |
| `sub` | 14 / 22 | 400 | 저자·설명 (밀리 최다 198회) |
| `caption` | 13 / 20 | 400 | 메타·리뷰 수 |
| `micro` | 12 / 18 | 500 | 뱃지 |

- **weight 3단계만**: 400 / 500 / 700. (교보 400·500·700 / 밀리 400·600·700 — 둘 다 3종)
- **책 제목은 400이 아니라 500.** 교보의 최다 weight가 500(227회)인 이유 — 목록에서 제목이 먼저 잡힌다.
- 소수점 크기(`15.2px`) 박멸.
- **다행 텍스트는 line-height를 따로 준다.** 밀리는 토큰 자체를 분리했다 — `body14-single 14/18` vs `body14-multi 14/22`.

### 2.3 셰이프 — 그림자를 버리고 선으로

```css
/* 삭제 대상 */
.cover { box-shadow: 0 56px 96px -24px rgba(0,0,0,.3); }  /* ← 목업 느낌의 주범 */
<Card className="shadow-sm">                               /* ← 전면 제거 */
```

- **카드 그림자 0.** 대신 `1px solid var(--line)`.
  (교보 홈 그림자 **2개** / 밀리 **4개** / page0127 랜딩 **10개**)
- **표지 그림자 0.** 밀리 표지 254개 전수 확인 결과 그림자·테두리·3D·기울임 **전부 없음.** radius만 있다.
- 그림자는 **"실제로 떠 있어야 하는 것"에만** — 드롭다운, 모달, 토스트, sticky 헤더.

### 2.4 도메인 셰이프 — page0127의 "책"

교보는 `16px 16px 16px 0px`(책등), 밀리는 토큰에도 없는 `7px`을 표지에만 하드코딩했다. **둘 다 표지를 위해 시스템 밖으로 나갔다.** page0127도 자기만의 셰이프 하나를 가져야 한다.

```css
/* 표지 — 왼쪽이 책등이라 각지고, 오른쪽 모서리만 둥글다 */
.cover {
  border-radius: 2px 6px 6px 2px;
  border: none;
  box-shadow: none;
  /* 좌측 책등 — 종이 두께 */
  background-image: linear-gradient(
    to right,
    rgba(0,0,0,.14) 0px,
    rgba(0,0,0,.06) 3px,
    transparent 7px
  );
}
```

**그리고 판형을 크롭하지 않는다 — 밀리의 가장 중요한 발견**

```css
/* ❌ 지금: 모든 표지를 같은 비율로 강제 → "DB에서 일괄 생성한" 느낌 */
.cover { aspect-ratio: 1 / 1.44; object-fit: cover; }

/* ✅ 밀리 방식: width만 고정, height는 원본 판형대로 */
.cover { width: 120px; height: auto; }
```

밀리의 "밀리의 발견" 섹션은 표지 width만 125px로 고정하고 **height가 175~190px로 제각각**이다. 문고본과 양장본은 실제로 판형이 다르다. **그 불균일을 굳이 살려둔 것**이 "진짜 책을 진열한" 느낌을 만든다. 균일 크롭은 자동 생성의 냄새다.

> 단, 그리드 정렬이 필요한 화면(내 서재 그리드)은 `align-items: end`로 **아랫단을 맞춰** 책이 선반 위에 선 것처럼 보이게 한다.

### 3.1 현황 — 시각 자산이 사실상 0

| 자산 | 현재 |
| --- | --- |
| 로고 | **없음.** 텍스트 `page0127`만 |
| 파비콘 | **Next.js 기본 그대로** |
| `public/` | `create-next-app` 데모 SVG 5개 |
| 표지 플레이스홀더 | `/images/placeholder-cover.png` → **404** |

| 히어로 | **이미지 0개.** 텍스트만 |

**표지가 유일한 사진 자산인데, 그걸 "카드 안의 썸네일"로 취급하고 있다.**

### 3.2 원칙 — 표지가 주인공이다

**① 히어로를 책장으로 바꾼다**
지금 히어로는 50px 텍스트 + 버튼 = 이미지 0개. 밀리 히어로는 1392×400 배너 **이미지**, 교보는 32슬라이드 캐러셀이다.
히어로 배경을 **실제 유저의 책장(표지 그리드)** 으로 깔고 그 위에 짧은 카피를 얹는다.
**"책장을 보면 그 사람이 보인다"는 카피는 책장을 보여줄 때만 성립한다.**

**② 알라딘 책등(spineflip) 이미지 — 우리만의 자산**
`src/shared/lib/imageUtils.ts`에 **이미 책등 URL 변환 로직이 구현돼 있다.**
```
https://image.aladin.co.kr/product/12345/67/spineflip/1234567890_d.jpg
```
교보도 밀리도 이걸 화면에 안 쓴다. **책을 세워서 꽂은 진짜 책장 뷰**를 만들 재료가 이미 손에 있다. 공개 서재(`/[username]`)를 "표지 그리드 ↔ 책등 책장" 토글로 만들면 그 자체가 서비스의 시그니처가 된다.

**③ 표지 없는 책 = 생성 표지** (404 대신)
제목·저자를 조판한 표지. 배경 `--bg-sunken`, 제목 14px/700 좌측 정렬 3줄 clamp, 저자 12px `--text-3`, 좌측 책등 그라디언트 동일 적용.

**④ 아바타 = 모노그램** — 닉네임 첫 글자 + 해시 기반 배경색. 회색 사람 실루엣 금지.

**⑤ OG 이미지 재제작** — 그라디언트+이모지 → 실제 표지 그리드. 공유했을 때 **책이 보여야** 한다.

### 3.3 금지 목록

| 금지 | 이유 |
| --- | --- |
| 스톡 사진 (창가에서 웃으며 책 읽는 사람) | AI 랜딩 클리셰 2순위 |
| 3D 렌더 일러스트 / 글래스모피즘 | 두 벤치마크 모두 0 |
| **이모지 아이콘** | 전량 lucide 단색 SVG로 |
| 그라디언트 배경 (`from-slate-50 via-blue-50 to-indigo-50`) | `PageContainer` gradient variant 폐기 |
| 표지 드롭섀도 · 균일 크롭 | 2.3 / 2.4 |

## 4. 콘텐츠 구성 (IA) — 가장 큰 구멍

### 4.1 비로그인 방문자는 **아무것도 볼 수 없다**

배포된 랜딩의 링크를 전수 조사한 결과:

```
/   /login   /about   /privacy   /terms   /contact
```

**콘텐츠로 이어지는 링크가 0개다.** 게다가

- `/about` `/privacy` `/terms` `/contact` → **전부 "🚧 준비 중" 스텁**
- 랜딩의 책 랭킹 카드는 **링크조차 아니다** — 클릭해도 아무 일도 안 일어난다
- 방문자가 할 수 있는 유일한 행동은 **로그인**

교보·밀리는 비로그인 방문자에게 콘텐츠를 **전부 연다**. 로그인은 담아둘 때 필요하지 구경할 때 필요한 게 아니다. 지금 page0127은 **문 앞에 벽을 세워두고 "들어와 보세요"라고 쓴 상태**다.

### 4.2 공개 영역을 연다

| 라우트 | 현재 | 변경 |
| --- | --- | --- |
| `/books/all` | protected | **공개** — 서비스의 카탈로그 |
| `/books/info/[id]` | protected | **공개** — "이 책을 읽은 사람들", 평균 별점. **SEO 자산 1순위** |
| `/[username]` | 이미 공개 | 랜딩에 **진입 링크 신설** (지금 입구가 없다) |
| 랜딩 랭킹 카드 | 링크 아님 | `/books/info/[id]` 로 링크 |

### 4.3 랜딩을 "마케팅 페이지"에서 "서비스의 첫 화면"으로

교보 홈은 폴드 안에 상품 **21개**가 보인다. page0127 랜딩은 폴드에 **책 0권**이다.

```
[헤더]  page0127          전체 도서   서재 둘러보기        로그인  시작하기

[히어로 — 축소, 배경은 실제 표지 그리드]
   책장을 보면, 그 사람이 보인다          ← 28px (기존 50px)
   읽은 책을 기록하면, 취향이 보입니다.    ← "당신" 제거
   [내 책장 만들기]                       ← 좌측 정렬

[이번 주 많이 읽힌 책]      2026.07.12 기준        전체보기 →
   표지 6~8권 · 각 카드에 ▲순위변동 · N명 완독

[방금 채워진 책장]                                전체보기 →
   최근 공개 서재 4개 · 책등 미리보기 + 닉네임 + 권수
   → /[username] 로 (지금 없는 진입로)

[읽고 나서 남긴 말]                               ← UGC, 무편집
   최근 메모·별점 3~4개 (공개 설정된 것만)

[독서 궁합]  두 사람의 책장을 나란히 놓으면
   실제 궁합 결과 1개 (지금의 정적 목업 대신)

[푸터]  소개 · 개인정보처리방침 · 이용약관 · 문의
        만든 사람 · 업데이트 로그 · 문의 이메일
```

**핵심: 폴드 안에 책이 보여야 한다.** 지금은 "이 서비스가 무엇인지 **설명하는** 페이지"고, 바꿀 것은 "무엇인지 **보여주는** 페이지"다.

### 4.4 큐레이션 축 — 밀리는 9가지로 묶는다

page0127은 지금 "완독 수 랭킹" **한 축**뿐이다. 밀리의 축을 참고해 우리가 **데이터로 만들 수 있는 것**만 고른다.

| 밀리의 축 | 원문 예시 | page0127 적용 가능성 |
| --- | --- | --- |
| **데이터/랭킹** | "밀리 랭킹" (9탭, 일간, 기준일 명시) | ✅ 이미 있음 — **기준일·변동만 추가** |
| **사람(회원)** | "오늘의 인생책" (회원 1인 추천) | ✅ **"오늘의 인생책 — {닉네임}님의 10점"** |
| **감정·상황 처방** | "이럴 땐 이런 책" — "지치고 힘들 때 위안이 되는 책" | ✅ 취향 분석 태그를 재활용 |
| **계절/제철** | "내가 '여름'에게로 간다" (제목에 '여름' 든 책만 모은 말장난) | ✅ 제목 매칭이면 쿼리로 가능 |
| **에디터** | "요즘 책 뭐 봄?" | △ 1인 운영이면 **오히려 강점** (아래 5.1 참조) |
| 명사 PICK / IP관 / 오리지널 / AI | "민규동·변영주 PICK" | ✗ 지금은 불가 |

> 밀리의 "내가 '여름'에게로 간다"는 **의미론적으로 엉성한 큐레이션**이다 — 그냥 제목에 '여름'이 든 책을 모았다. 그런데 **사람이 낄낄대며 짠 티가 난다.** 이게 알고리즘 추천보다 사람 냄새를 낸다.

### 4.5 "준비 중" 4페이지를 실제로 채운다

`/about` `/privacy` `/terms` `/contact` 가 전부 🚧 스텁인 건 **디자인 문제가 아니라 신뢰 문제**다. 개인정보처리방침은 소셜 로그인 + 개인 데이터를 다루는 이상 **법적 의무**이기도 하다.

- `/about` — 왜 만들었는지, **누가 만들었는지**. 만든 사람이 보이는 게 "운영되는 서비스"의 증거다.
- `/privacy` `/terms` — 실제 내용. 로그인 화면의 약관 링크가 지금 `href="#"`(죽은 링크)인 것도 함께 수정.
- `/contact` — 이메일 하나면 충분하다.

## 5. 카피라이팅

### 5.1 원칙 8개 — 밀리 실측에서 도출

**① "당신"을 지운다.**
밀리는 홈 전체에서 "당신"을 **3개 캠페인, 4회**만 쓴다. **섹션 제목·UI 문구엔 0회.** 주어를 생략하거나 `{닉네임}님`.

**② 화자를 세운다 — 가장 중요.**
"밀리가 주목한 책", "MD들이 신중하게 골랐어요". 주어가 **사용자가 아니라 화자**다.
page0127의 화자는 누구인가? **개인 프로젝트라는 게 오히려 무기다.** "page0127이" 또는 만든 사람 본인이 화자가 될 수 있다. 대기업이 흉내 못 내는 자산이다.
- `이번 주 많이 읽힌 책` (무주어) → 그대로 두되,
- `제가 골라둔 책` / `page0127이 눈여겨본 책` 같은 **화자 있는 섹션을 하나 만든다.**

**③ 숫자와 고유명사를 문장의 뼈대로.**
밀리: "김초엽부터 신형철까지", "별점 4.5 이상", "특별 초대권 100장", "하루 15분씩".
우리: "많은 분들이 끝까지 읽어낸"(✗) → **"이번 달 27명이 완독했어요"**(○)

**④ 날짜를 박는다.** `2026.07.12 기준` · `07월 13일 월요일 업데이트`. 갱신 책임을 지겠다는 선언이다.

**⑤ 어투를 섞는다 — 균질함이 AI의 냄새다.**
밀리는 **반말 의문**("요즘 책 뭐 봄?") + **감탄·의성어**("따끈따끈", "쉿!", "지금 읽어야 제맛!") + **존댓말 CTA**("첫 달 무료로 즐겨보세요")를 **한 화면에서 섞는다.**
지금 page0127은 전부 "~해요/~습니다"로 균질하다. 톤 자체는 좋으나 **리듬이 하나**다.

**⑥ 길이 분산을 허용한다.**
밀리 UGC 답변 길이: **7자**("엄청 슬퍼요ㅠ") ~ **393자**. AI 생성 카피는 15~25자로 균질하다.

**⑦ UGC를 다듬지 않는다 — 강력한 증거.**
밀리는 회원 답변·닉네임의 오탈자를 **그대로 노출**한다:
> `"금새 완독입니다"`(금세✗) · `"이야기는 시작되요"`(시작돼요✗) · `"설레서 머리 빡빡 치고 싶어요요"` · `"이거 읽으면 밀리를 못 지나치게 될 거야,,,"`
> 닉네임: `"뜨개을 좋아하는 솜베개_130702"` — 오타 + **자동생성 suffix까지 그대로**

**다듬지 않았다는 게 곧 사람이 썼다는 증거다.** page0127도 유저 메모를 그대로 노출하고, 짧은 메모("좋았다")도 숨기지 말 것.

**⑧ 이모지 헤딩 폐지.** lucide 단색 아이콘 또는 아이콘 없음.

> **보너스 — AI가 절대 못 쓰는 두 문장**
> - **자기부정**: 밀리 `"알고리즘보다 정확한 밀리 회원 추천: 6월"` — 자기 알고리즘을 스스로 깎아내린다.
> - **나쁜 소식**: 밀리 `"첫 달 무료 혜택 종료 사전 안내_08/04(화)"` — 혜택 종료 예고를 홈에 띄운다.
>
> page0127도 쓸 수 있다: **"AI 분석이 늘 맞진 않아요. 틀렸다면 알려주세요."** / **"7월 3일 — 궁합 분석이 자꾸 실패해서 고쳤습니다."**

## 6. 기능 — "운영의 흔적"을 만든다

디자인·카피를 다 고쳐도 이게 없으면 여전히 목업이다.

| # | 기능 | 근거 | 난이도 |
| --- | --- | --- | --- |
| 1 | **순위 변동** `▲12` `▼3` `NEW` | 교보 `▲448`, 밀리 `▲57`. **어제 스냅샷이 DB에 있어야** 계산 가능 → 그 자체가 운영 중의 증거 | 중 (스냅샷 테이블 + cron) |
| 2 | **집계 기준일** `2026.07.12 기준` | 교보·밀리 둘 다 명시 | 하 |
| 3 | **요일까지 쓴 갱신 표기** `07월 13일 월요일 업데이트` | 밀리 | 하 |
| 4 | **상대 시간** `3시간 전 완독` | 활동 피드·랜딩 | 하 |
| 5 | **오늘의 숫자** `오늘 4권이 완독됐어요` | 매일 바뀌는 문자열이 최소 하나는 있어야 한다 | 하 |
| 6 | **검색 placeholder 회전** | 교보는 실제 편집 카피 3개가 돈다 | 하 |
| 7 | **빈 데이터를 정직하게** `리뷰(1)` | 교보는 리뷰 1개인 책을 그대로 노출 | 하 |
| 8 | **업데이트 로그** (`/about` 하단) | "7월 3일 — 궁합 분석 오류 수정". **개인 프로젝트일수록 강력** | 하 |
| 9 | **서피스별 정보 밀도 분리** | 아래 부연 | 중 |

**9번 부연 — 정보 밀도를 면마다 다르게**

교보 홈의 베스트 카드는 `표지 + 순위 + ▲변동 + 제목` **4필드뿐**이다. **가격도 저자도 없다.** 반면 베스트셀러 리스트 카드는 516×236px 안에 텍스트 필드가 **24개**다.
