# Supabase 가이드 (프론트엔드 개발자 관점)

> **핵심**: Supabase는 프론트엔드 개발자가 백엔드 없이 앱을 만들 수 있게 해주는 서비스입니다.

---

## 📌 Supabase란?

**"Firebase의 오픈소스 대안"**

- PostgreSQL 데이터베이스 + 인증 + 스토리지 + Realtime을 한 번에 제공
- 프론트엔드 개발자가 **JavaScript만으로** 백엔드 기능 사용 가능
- 무료 티어로 충분히 시작 가능

---

## 🚀 설치

```bash
npm install @supabase/supabase-js @supabase/ssr
```

- `@supabase/supabase-js`: Supabase 클라이언트 라이브러리 (필수)
- `@supabase/ssr`: Next.js App Router용 SSR 헬퍼 (Next.js 사용 시 필수)

## 🎯 프론트엔드 개발자가 알아야 할 핵심 개념

### 1. 왜 3가지 클라이언트가 필요한가?

Next.js App Router에서는 코드가 **3가지 환경**에서 실행됩니다:

| 환경                  | 실행 위치 | 파일 위치                    | 용도                    |
| --------------------- | --------- | ---------------------------- | ----------------------- |
| **Server Component**  | 서버      | `lib/supabase/server.ts`     | 기본 데이터 조회        |
| **Client Component**  | 브라우저  | `lib/supabase/client.ts`     | 버튼 클릭, 폼 제출 등   |
| **Middleware**        | Edge      | `lib/supabase/middleware.ts` | 페이지 접근 전 인증체크 |

#### 왜 분리해야 하나?

- **쿠키 처리 방식이 다름**: 서버/브라우저/Edge에서 쿠키 접근 방법이 달라서
- **보안**: 각 환경에 맞는 안전한 인증 처리 필요

---

### 2. Server Component vs Client Component

Next.js App Router의 핵심 개념입니다.

#### Server Component (기본)

```tsx
// app/page.tsx
import { createClient } from '@/lib/supabase/server'

export default async function Page() {
  const supabase = await createClient()

  // 서버에서 데이터 조회 (빠르고 안전)
  const { data } = await supabase
    .from('reading_records')
    .select('*')

  return <div>{/* data 사용 */}</div>
}
```

✅ **장점:**
- 데이터베이스와 직접 통신 (빠름)
- API 키가 브라우저에 노출 안 됨 (안전)
- SEO 좋음 (HTML에 데이터 포함)

❌ **제약:**
- `useState`, `useEffect` 사용 불가
- 버튼 클릭 등 이벤트 처리 불가

#### Client Component ('use client')

```tsx
'use client'
// app/components/LikeButton.tsx
import { createClient } from '@/lib/supabase/client'
import { useState } from 'react'

export default function LikeButton() {
  const [liked, setLiked] = useState(false)
  const supabase = createClient()

  const handleLike = async () => {
    // 브라우저에서 데이터 추가
    await supabase.from('likes').insert({ ... })
    setLiked(true)
  }

  return <button onClick={handleLike}>좋아요</button>
}
```

✅ **장점:**
- `useState`, `onClick` 등 React 훅 사용 가능
- 인터랙티브한 기능 구현 가능

❌ **제약:**
- 브라우저에서만 실행 (초기 로딩 느림)

### 3. Middleware는 왜 필요한가?

**문제 상황:**
- 로그인 안 한 사용자가 `/dashboard`에 접근하면?
- 페이지가 로딩된 후에야 "로그인하세요" 메시지를 보여줌 (나쁜 UX)

**Middleware 해결책:**
- 페이지 렌더링 **전에** 인증 체크
- 로그인 안 했으면 즉시 `/login`으로 리디렉션

```typescript
// middleware.ts
export async function middleware(request: NextRequest) {
  const { user } = await supabase.auth.getUser()

  // 인증 안 되었으면 로그인 페이지로 리디렉션
  if (!user && !request.nextUrl.pathname.startsWith('/login')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  return NextResponse.next()
}
```

## 📚 SQL 초기 설정 (한 번만 하면 됨)

Supabase 대시보드 → SQL Editor에서 실행

### 왜 SQL이 필요한가?

- **테이블 생성**: 책, 사용자 등 데이터 구조 정의
- **보안 규칙(RLS)**: "내 데이터만 볼 수 있게" 설정
- **자동화**: 회원가입 시 프로필 자동 생성 등

### 핵심 개념 3가지

#### 1. Row Level Security (RLS)

```sql
-- "본인만 자신의 기록 조회 가능"
CREATE POLICY "Users can view own records"
  ON reading_records FOR SELECT
  USING (auth.uid() = user_id);
```

- **백엔드 없이 보안 구현**: SQL로 "누가 어떤 데이터를 볼 수 있는지" 정의
- 프론트엔드에서 아무리 해킹해도 다른 사람 데이터 못 봄

#### 2. Trigger & Function

```sql
-- Auth 가입 시 users 테이블에 자동 생성
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW
  EXECUTE FUNCTION handle_new_user();
```

- Google 로그인 → 자동으로 `users` 테이블에 프로필 생성
- 백엔드 코드 없이 자동화

#### 3. Foreign Key

```sql
user_id UUID REFERENCES users(id) ON DELETE CASCADE
```

- 사용자 삭제 → 관련 독서 기록도 자동 삭제
- 데이터 정합성 유지

## 💡 프론트엔드에서 사용하는 Supabase JS

SQL은 초기 설정만! 이후는 **JavaScript만** 사용합니다.

### CRUD 예제

```typescript
// 조회 (SELECT)
const { data } = await supabase
  .from('reading_records')
  .select('*')
  .eq('user_id', userId)

// 추가 (INSERT)
await supabase
  .from('reading_records')
  .insert({ book_title: '1984', status: 'reading' })

// 수정 (UPDATE)
await supabase
  .from('reading_records')
  .update({ status: 'completed' })
  .eq('id', recordId)

// 삭제 (DELETE)
await supabase
  .from('reading_records')
  .delete()
  .eq('id', recordId)
```

### SQL vs Supabase JS 비교

| SQL 개념 | 의미 | Supabase JS        |
| -------- | ---- | ------------------ |
| SELECT   | 조회 | `.select()`        |
| INSERT   | 추가 | `.insert()`        |
| UPDATE   | 수정 | `.update()`        |
| DELETE   | 삭제 | `.delete()`        |
| WHERE    | 조건 | `.eq()`, `.gt()` 등 |
