# 프로젝트 개요

> **Page 0127 (페이지 읽다)** - 개인 독서 기록 관리 및 시각화 서비스

---

## 📖 프로젝트명 의미

**Page 0127** → **Page 읽 .** (페이지 읽다)

- `01` + `27` = `읽` (한글 조합)
- `.` = `dot` (닷)
- "페이지를 읽다"라는 의미의 독서 기록 서비스

---

## 🎯 프로젝트 목표

### 개인 목표

1. **기술 습득**: Next.js 16, React Query, Supabase 등 최신 기술 스택 학습
2. **포트폴리오**: 실제 사용 가능한 프로덕트 개발 경험
3. **완성**: 4-5개월 내 MVP 배포 및 단계적 기능 확장

### 서비스 목표

- **문제 정의**: 독서 기록을 남기고 싶지만, 복잡하거나 SNS 위주 서비스는 부담스러움
- **솔루션**: 간단한 개인 책장과 시각화로 독서 기록의 즐거움 제공
- **타겟**: 20-40대 독서 애호가, 기록을 좋아하는 사람들

---

## 💡 핵심 차별화 포인트

| 구분          | 기존 서비스       | Page 0127             |
| ------------- | ----------------- | --------------------- |
| **포커스**    | SNS/커뮤니티 중심 | **개인 기록 중심**    |
| **UI**        | 복잡한 기능들     | **직관적이고 간결함** |
| **통계**      | 단순 권수 카운트  | **시각화 차트 제공**  |
| **추가 기능** | 커뮤니티 필수     | **AI 분석 (선택)**    |

### 서비스 특징

- ✅ **개인 중심**: SNS 부담 없이 나만의 책장 관리
- ✅ **시각화**: 월별 독서량, 카테고리별 차트로 한눈에 파악
- ✅ **간결함**: 복잡한 기능 없이 필요한 것만
- ✅ **확장성**: 원하면 AI 분석, SNS 기능 추가 가능

## 📅 개발 로드맵

### Phase 1: MVP (Week 1-6) 🥇

**목표**: 개인용 독서 기록 + 통계 시각화 완성

```
핵심 기능:
├─ Google 소셜 로그인
├─ 알라딘 API 책 검색
├─ 독서 기록 CRUD (읽고 싶은/읽는 중/완독)
├─ 개인 책장 (상태별 탭, 정렬, 필터)
├─ 독서 통계 대시보드
│   ├─ 총 권수/쪽수 표시
│   ├─ 월별 독서량 차트 (Bar Chart)
│   ├─ 카테고리별 레이더 차트
│   └─ 독서 진행률 (연간 목표 대비)
└─ 기본 프로필 설정
```

**예상 시간**: 6주
**마일스톤**: Vercel 배포 + 본인이 실사용
**완성 기준**: 첨부 이미지 화면 구현

---

### Phase 2: AI 취향 분석 (Week 7-10) 🤖

**목표**: 선택적 AI 분석 기능 추가

```
AI 기능:
├─ OpenAI 취향 분석
│   ├─ 독서 성향 분석 (성격, 패턴)
│   ├─ 선호 장르/키워드 추출
│   └─ 분석 결과 시각화
├─ AI 책 추천 (개인 맞춤)
├─ 무료 사용자 월 3회 제한
└─ 프리미엄 플랜 고려
```

**예상 시간**: 4주
**마일스톤**: AI 분석 만족도 80%+

---

### Phase 3: 소셜 네트워크 (Week 11-14) 👥

**목표**: 선택적 SNS 기능 (부가 기능)

```
소셜 기능:
├─ 팔로우/팔로워 시스템
├─ 피드 (타임라인)
├─ 댓글 & 좋아요
├─ 책 공개/비공개 설정
└─ 기본 알림
```

**예상 시간**: 4주
**마일스톤**: 베타 테스터 10명 확보

---

### Phase 4: 고급 통계 & 게이미피케이션 (Week 15-18) 📊

**목표**: 장기 사용 동기 부여

```
추가 기능:
├─ 독서 캘린더 (GitHub 히트맵)
├─ 독서 챌린지 & 목표 설정
├─ 연간 독서 리포트 (PDF)
├─ 뱃지/업적 시스템
└─ 랭킹 (선택)
```

**예상 시간**: 4주
**마일스톤**: 일 평균 방문 증가

---

### Phase 5: 폴리싱 & 최적화 (Week 19-22) ✨

**목표**: 완성도 향상 및 런칭 준비

```
최종 마무리:
├─ UI/UX 개선 (애니메이션, 반응형)
├─ 성능 최적화 (이미지, 번들 사이즈)
├─ SEO 최적화
├─ 접근성(a11y) 개선
└─ 다크 모드 (선택)
```

**예상 시간**: 4주
**마일스톤**: 정식 런칭

## 🚫 하지 않을 것 (명확한 범위 설정)

### MVP에서 제외

- ❌ SNS 기능 (Phase 3 이후)
- ❌ AI 분석 (Phase 2 이후)
- ❌ 복잡한 필터링 (장르/연도별만)
- ❌ 이메일 알림
- ❌ 다크 모드 (Phase 5)

### 완전히 제외

- ❌ 전자책 뷰어
- ❌ 책 구매 기능 (제휴 링크만 고려)
- ❌ 실시간 채팅
- ❌ 모바일 앱 (웹 반응형으로 충분)

### 제외 이유

- **집중**: 개인 독서 기록이라는 핵심 가치에 집중
- **범위**: 5-6개월 안에 완성 가능한 범위로 제한
- **차별화**: 간결함이 곧 차별화

### vs 독서 기록 앱

- ✅ **통계 시각화**로 동기부여
