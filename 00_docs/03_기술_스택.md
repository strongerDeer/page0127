# 기술 스택 가이드

> 선택한 기술과 그 이유

---

## Frontend

### Next.js 14 (App Router)

**선택 이유:**

- ✅ React 기반 풀스택 프레임워크
- ✅ App Router로 최신 패턴 학습
- ✅ SSR/SSG로 SEO 최적화
- ✅ Vercel 배포 최적화

**대안:**

- ❌ Vite + React: CSR만 지원
- ❌ Remix: 레퍼런스 부족

### TypeScript

**선택 이유:**

- ✅ 타입 안정성
- ✅ IDE 자동완성
- ✅ 리팩토링 용이

### Tailwind CSS + shadcn/ui

**선택 이유:**

- ✅ 유틸리티 우선 CSS
- ✅ shadcn/ui로 빠른 UI 구축
- ✅ 커스터마이징 쉬움

**대안:**

- ❌ MUI: 무겁고 커스터마이징 어려움
- ❌ Styled-components: 런타임 오버헤드

### React Query

**선택 이유:**

- ✅ 서버 상태 관리 최적
- ✅ 캐싱, 리페칭 자동화
- ✅ 낙관적 업데이트 지원

### Zustand

**선택 이유:**

- ✅ 간단한 클라이언트 상태 관리
- ✅ Redux보다 가볍고 쉬움
- ✅ TypeScript 지원 우수

### axios

**선택 이유:**

- ✅ RESTful API 호출에 최적화
- ✅ 인터셉터로 공통 로직 처리 (인증, 에러 핸들링)
- ✅ TypeScript 지원 우수
- ✅ 요청/응답 변환 자동화

**대안:**

- ❌ fetch: 인터셉터 없음, 보일러플레이트 많음
- ❌ Supabase SDK 직접 사용: DB 종속성 높음

## Backend & Database

### Supabase

**선택 이유:**

- ✅ PostgreSQL + Auth + Storage 통합
- ✅ 무료 티어 충분
- ✅ Next.js 통합 쉬움
- ✅ Realtime 기능 (알림용)

**대안:**

- ❌ Firebase: NoSQL 제약
- ❌ PlanetScale: Auth 별도 구현 필요

### 알라딘 API

**선택 이유:**

- ✅ 국내 도서 정보 풍부
- ✅ 무료 사용 가능
- ✅ 이미지 URL 제공

## AI & 외부 API

### OpenAI API (GPT-4)

**선택 이유:**

- ✅ 강력한 텍스트 분석
- ✅ 한국어 지원 우수
- ✅ JSON 모드 지원

**비용:**

- GPT-4: ~$0.03/1K tokens
- 월 예상: $10-20 (베타)

**대안:**

- ❌ Google Gemini: 한국어 성능 부족
- ❌ Claude: API 비용 비쌈
